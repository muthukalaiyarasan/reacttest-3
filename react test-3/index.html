1.Which of the following scenarios is a suitable use case for using the componentWillUpdate lifecycle method?

Answer: To perform validation of incoming props before rendering

2. In React, what is the purpose of the componentWillUpdate lifecycle method?

Answer: To perform actions before the component is re-rendered due to state or props changes.

3.In a React class component, where should you perform asynchronous data fetching and update the component's state based on the fetched data?

Answer: In the componentDidMount method.

4.How can you update component state in React without using the this.setState method?

Answer: It is not recommended to update state without this.setState.

5.How can you update component state in a functional component in React?

Answer: By calling the useState hook with new values.

6. In React, what is the primary purpose of the componentDidMount lifecycle method?

Answer: To perform initial setup and data fetching after the component is added to the DOM.

7.In a class component, which lifecycle method is used for fetching data from an API?

Answer: componentDidMount

8.How can you access the current state of a React component within its render method?

Answer: By referencing this.state

9.In React, what is the primary purpose of the componentDidUpdate lifecycle method?

Answer: To perform side effects after the component has re-rendered due to state or props changes.

10.In React, what is the primary use case for the componentWillUnmount lifecycle method?

Answer: To perform cleanup operations before the component is removed from the DOM.

11.When using the map function to render a list of items in React, why is it important to provide a unique key prop to each rendered element?

Answer: It helps React efficiently identify which elements have changed, been added, or been removed when updating the list.

12.What happens when you modify the state directly without using the setState method in a React component?

14.What is the key difference between state and props in a React component?

Answer: State is immutable, while props can change.

15.In React, what is a controlled component?

Answer: A component that receives data through props and notifies changes using callbacks.

16.Which lifecycle method is commonly used to initialize state or perform other setup operations in a React component?

Answer: componentDidMount

17.What is the purpose of the constructor method in a React class component?

Answer: To initialize component state and bind event handlers.

18.How do you pass data from a child component to a parent component in React?

Answer: By calling a callback function passed as a prop from the parent to the child component.

19.Which of the following is NOT a valid way to set the initial state of a React component?

Answer: Using the componentWillMount lifecycle method

20. How can you pass data from a parent component to a child component in React using props?

Answer: By defining a prop in the child component's JSX element when rendering it in the parent component
